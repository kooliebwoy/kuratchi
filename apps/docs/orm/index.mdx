## SQLite for the runtime you already have

`@kuratchi/orm` is a lightweight ORM for Cloudflare D1 and Durable Object SQLite. It keeps the API Worker-safe, uses a compact schema DSL, and stays close to SQL instead of hiding the database behind a large abstraction layer.

## What it gives you

<CardGroup cols={2}>
  <Card title="Schema DSL" icon="table-columns" href="/orm/schema-dsl">
    Define tables, mixins, indexes, JSON columns, enums, and foreign keys with a compact object format.
  </Card>
  <Card title="Chainable queries" icon="magnifying-glass" href="/orm/querying">
    Query with `where`, `orderBy`, `include`, `limit`, `offset`, `count`, and `distinct` without leaving SQL-shaped concepts behind.
  </Card>
  <Card title="D1 and DO support" icon="database" href="/orm/getting-started">

  </Card>
</CardGroup>

## Design constraints

The ORM is built around a few hard constraints:

- No Node-only runtime assumptions
- No dependency-heavy query layer
- SQL semantics stay visible
- Same mental model across D1 and Durable Object SQLite
