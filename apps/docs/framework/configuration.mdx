## `kuratchi.config.ts`

The config file is optional. You only need it when wiring framework integrations such as ORM, auth, CSS processing, or custom asset settings.

```ts
import { defineConfig } from '@kuratchi/js';
import { kuratchiOrmConfig } from '@kuratchi/orm/adapter';
import { kuratchiAuthConfig } from '@kuratchi/auth/adapter';

export default defineConfig({
  orm: kuratchiOrmConfig({
    databases: {
      DB: { schema: appSchema },
      NOTES_DO: { schema: notesSchema, type: 'do' },
    },
  }),
  auth: kuratchiAuthConfig({
    cookieName: 'kuratchi_session',
    sessionEnabled: true,
  }),
});
```

Without `kuratchi.config.ts`, the compiler falls back to framework defaults.

## Static assets

For plain CSS, images, and other static files, put them in `src/assets`.

Kuratchi mirrors `src/assets` into the generated public assets directory and keeps Wrangler's `assets.directory` in sync automatically, so the default public URL is `/assets/...`.

```html
<link rel="stylesheet" href="/assets/app.css" />
```

If you want a different public prefix, set `assetsPrefix`:

```ts
import { defineConfig } from '@kuratchi/js';

export default defineConfig({
  assetsPrefix: '/static/',
});
```

Then reference the same file with the new prefix:

```html
<link rel="stylesheet" href="/static/app.css" />
```

## CSS Configuration

CSS processing is opt-in. Install the packages you need and configure in `kuratchi.config.ts`.

### CSS Minification

Install Lightning CSS for production minification:

```bash
npm install lightningcss
```

Minification is automatic in production builds — no config needed.

### Tailwind CSS

Install the required packages:

```bash
npm install tailwindcss @tailwindcss/postcss postcss
```

Enable Tailwind in your config:

```ts
import { defineConfig } from '@kuratchi/js';

export default defineConfig({
  css: {
    tailwind: true,
  },
});
```

Create your CSS file with Tailwind imports:

```css
/* src/assets/app.css */
@import "tailwindcss";
```

### Tailwind Plugins (DaisyUI, Forms, etc.)

Add plugins with the `plugins` array:

```bash
npm install daisyui @tailwindcss/forms
```

```ts
export default defineConfig({
  css: {
    tailwind: true,
    plugins: ['daisyui', '@tailwindcss/forms'],
  },
});
```

Reference plugins in your CSS:

```css
/* src/assets/app.css */
@import "tailwindcss";
@plugin "daisyui";
@plugin "@tailwindcss/forms";
```

Then use the classes in your templates:

```html
<button class="btn btn-primary">Click me</button>
<div class="card bg-base-100 shadow-xl">
  <div class="card-body">
    <h2 class="card-title">Card Title</h2>
  </div>
</div>
```

## @kuratchi/ui Component Library

The `@kuratchi/ui` package provides server-rendered HTML components. It's independent of your CSS choice.

```ts
import { defineConfig } from '@kuratchi/js';
import { kuratchiUiConfig } from '@kuratchi/ui/adapter';

export default defineConfig({
  ui: kuratchiUiConfig({ 
    theme: 'dark',    // 'light' | 'dark' | 'system'
    radius: 'default' // 'none' | 'default' | 'full'
  }),
});
```

Use components in your routes:

```html
<script>
  import Card from '@kuratchi/ui/card.html';
  import Badge from '@kuratchi/ui/badge.html';
</script>

<Card>
  <Badge variant="success">Active</Badge>
  <p>Card content</p>
</Card>
```

## What belongs here

Use the config file for:

- CSS processing (Tailwind, minification)
- ORM schema bindings
- Auth configuration
- UI component theming

Routing, Durable Object discovery, and generated worker output are handled by the compiler.
