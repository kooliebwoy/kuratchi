## `kuratchi.config.ts`

The config file is optional. You only need it when wiring framework integrations such as UI, ORM, or auth.

```ts
import { defineConfig } from '@kuratchi/js';
import { kuratchiUiConfig } from '@kuratchi/ui/adapter';
import { kuratchiOrmConfig } from '@kuratchi/orm/adapter';
import { kuratchiAuthConfig } from '@kuratchi/auth/adapter';

export default defineConfig({
  ui: kuratchiUiConfig({ library: 'tailwindcss' }),
  orm: kuratchiOrmConfig({
    databases: {
      DB: { schema: appSchema },
      NOTES_DO: { schema: notesSchema, type: 'do' },
    },
  }),
  auth: kuratchiAuthConfig({
    cookieName: 'kuratchi_session',
    sessionEnabled: true,
  }),
});
```

Without `kuratchi.config.ts`, the compiler falls back to framework defaults.

## UI Configuration

The `ui` config supports Tailwind CSS and plugins like DaisyUI.

### Tailwind CSS

Enable Tailwind CSS with the `library` option:

```ts
ui: kuratchiUiConfig({ 
  library: 'tailwindcss'
})
```

The framework automatically:
- Detects `@tailwindcss/cli` in your project
- Generates the input CSS with `@import "tailwindcss"`
- Builds and injects the output CSS into your layout

### Tailwind Plugins (DaisyUI, Forms, etc.)

Add plugins with the `plugins` array:

```ts
ui: kuratchiUiConfig({ 
  library: 'tailwindcss',
  plugins: ['daisyui', 'forms']
})
```

**Supported plugins:**
- `daisyui` — Component library with themes
- `forms` — Better form styling (expands to `@tailwindcss/forms`)
- Any npm package name that's a valid Tailwind plugin

**Example with DaisyUI:**

```ts
// kuratchi.config.ts
import { defineConfig } from '@kuratchi/js';
import { kuratchiUiConfig } from '@kuratchi/ui/adapter';

export default defineConfig({
  ui: kuratchiUiConfig({ 
    library: 'tailwindcss',
    plugins: ['daisyui']
  }),
});
```

Then use DaisyUI classes in your templates:

```html
<button class="btn btn-primary">Click me</button>
<div class="card bg-base-100 shadow-xl">
  <div class="card-body">
    <h2 class="card-title">Card Title</h2>
    <p>Card content</p>
  </div>
</div>
```

### @kuratchi/ui Theme Options

If you're using the `@kuratchi/ui` component library, you can also configure theme options:

```ts
ui: kuratchiUiConfig({ 
  theme: 'dark',    // 'light' | 'dark' | 'system'
  radius: 'default' // 'none' | 'default' | 'full'
})
```

## What belongs here

Use the config file for:

- UI adapter setup (Tailwind, plugins, themes)
- ORM schema bindings
- Auth configuration

Routing, Durable Object discovery, and generated worker output are handled by the compiler.
