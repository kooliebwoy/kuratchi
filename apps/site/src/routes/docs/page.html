KuratchiJS LLM Docs

Optimized for machine consumption. This page summarizes the framework contract from README.

Install and boot

  • Install: npm install @kuratchi/js
  • Create: npx kuratchi create my-app
  • Dev: bun run dev
  • Build: npx kuratchi build

Routing contract

  • src/routes/page.html -> /
  • src/routes/items/page.html -> /items
  • src/routes/blog/[slug]/page.html -> /blog/:slug
  • src/routes/layout.html wraps all routes and must include <slot></slot>

Actions and forms

  • Exported functions referenced by action= become server actions.
  • Actions receive FormData.
  • Use ActionError for user-facing validation failures.
  • Use redirect() for POST-Redirect-GET flows.

Durable Object RPC

  • Any .do.ts file is treated as a Durable Object handler source.
  • Function mode is recommended: exported functions become RPC methods.
  • Class mode supported via kuratchiDO.
  • Client/runtime imports use generated $durable-objects/* proxy modules.

Config integrations

  • orm: D1 and DO SQLite schemas via @kuratchi/orm
  • auth: credentials, oauth, guards, rate limiting, roles, turnstile, organizations via @kuratchi/auth
  • durableObjects: class names, optional stubId, and file mapping

Runtime APIs

Available server-side helpers include getCtx, getRequest, getLocals, getParams, getParam, redirect, and goto.
