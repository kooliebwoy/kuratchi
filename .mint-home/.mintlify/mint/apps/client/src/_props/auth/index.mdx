## Auth that stays inside the framework runtime

`@kuratchi/auth` is a config-driven auth package for Kuratchi apps. It is built around Worker runtime constraints, D1 or DO-backed persistence, and compiler-integrated request hooks for session cookies, guards, rate limiting, and Turnstile.

## Package model

The package is split into focused runtime modules:

* credentials and sessions
* low-level auth context
* activity logging
* OAuth providers
* guards
* rate limiting
* Turnstile verification
* organization-aware multi-tenancy
